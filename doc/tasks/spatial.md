# 空间索引模块 - 最小可执行任务列表

## 文件：`src/spatial.py`

### 任务清单

- [ ] **S1. 实现环形宇宙工具函数**
  - `ring_distance()`：计算环形宇宙中两点最短距离，考虑 4 条路径
  - `normalize_position()`：将坐标标准化到 `[0, universe_size)` 范围
  - 编写单元测试验证跨边界距离计算正确性、坐标标准化正确性

- [ ] **S2. 定义 `SpatialIndex` 类核心数据结构**
  - `__init__`：接收 universe_size 和 cell_size，计算 grid_size
  - 一维数组模拟二维网格 `self._grid`
  - `self._id_to_cell` 字典用于快速查找
  - `_cell_index()`：坐标到网格一维索引映射（处理环形边界）
  - `_neighbor_cell_indices()`：返回覆盖半径内所有网格索引（处理环形边界）
  - 编写单元测试验证空索引、单文明索引的初始化和查询

- [ ] **S3. 实现 `rebuild()` 方法**
  - 清空网格数据
  - 遍历所有文明，根据坐标插入对应网格
  - 更新 `_id_to_cell` 映射
  - 复杂度 O(N)
  - 编写单元测试验证重建后索引正确性

- [ ] **S4. 实现 `query_neighbors()` 邻近查询**
  - 获取覆盖网格索引
  - 收集候选文明
  - 精确距离过滤（使用环形距离）
  - 不返回自身文明
  - 复杂度 O(k)，k = 覆盖网格内总文明数
  - 编写单元测试验证：范围内找到、范围外找不到、跨环形边界查询正确、自身排除

- [ ] **S5. 实现辅助方法**
  - `query_region()`：矩形区域查询（考虑环形边界）
  - `remove_civilization()`：从索引中移除指定文明
  - `cell_count` 属性：返回非空网格数
  - `load_balance_stats` 属性：返回网格负载统计（min/max/avg 每网格文明数、空网格比例）
  - 编写单元测试验证各辅助方法

- [ ] **S6. 实现 `auto_select_cell_size()` 自动网格尺寸选择**
  - 基于 avg_detection_range 的基础值（让查询覆盖约 3×3 网格）
  - 基于目标每网格文明数 10~50 的调整值
  - 取两者中间值，根据 CPU 评分微调
  - 结果限制在合理范围内
  - 编写单元测试验证不同场景的推荐值合理性

- [ ] **S7. 性能基准测试**
  - 编写 `pytest.mark.benchmark` 测试：创建 10000 个文明并查询
  - 验证耗时在可接受范围内（参考值：< 50ms）
  - 记录测试结果供后续优化参考
