## 1. 项目概述

### 1.1 核心理念

本模拟器以「黑暗森林」宇宙社会学为理论框架，将每个文明抽象为二维空间中的一个**点状实体**，模拟其诞生、发展、扩张、探测、接触、攻击与毁灭的完整生命周期。

### 1.2 核心假设

| 假设 | 说明 |
|------|------|
| **点状文明** | 每个文明占据一个空间坐标 (x, y)，不具有形状或面积 |
| **猜疑链** | 两个文明接触时，因无法判断对方善意，倾向于先发制人 |
| **技术爆炸** | 文明可能经历爆发式科技增长，大幅提升探测和攻击能力 |
| **暴露即死亡** | 文明坐标一旦暴露，可能招致毁灭性打击 |
| **宇宙打击** | 存在低概率的全局随机毁灭事件（黑暗森林打击） |
| **环形宇宙** | 宇宙边界在拓扑上连接（绕环），没有边界 |

## 2. 设计思路

### 2.1 三层架构

采用**三层架构**，下层为上层提供服务，层间通过显式参数传递交互：

```
┌─────────────────────────────────────────────────────┐
│                    应用层                             │
│  main.py / cli.py / batch.py                        │
│  命令行解析、模式选择、批处理调度                      │
├─────────────────────────────────────────────────────┤
│                    可视化层                            │
│  plotter.py / run_modes.py / interactive.py         │
│  实时图表、交互控制、文明详情查看                      │
├─────────────────────────────────────────────────────┤
│                    核心层                              │
│  config / entity / spatial / simulation              │
│  rules/  (tech_bomb, expansion, detection, DF)       │
│  output/ (stats, data_saver)                         │
└─────────────────────────────────────────────────────┘
```

### 2.2 设计原则

| 原则 | 体现 |
|------|------|
| **显式参数传递** | 所有模块间通过函数参数传递数据，禁止全局变量（Config 例外作为只读配置） |
| **Data 类为纯容器** | `Civilization`、`StepStats`、`SimEvent` 等数据类仅存数据，不含业务逻辑 |
| **松耦合规则模块** | 每个规则模块是独立函数，接收文明列表 + 空间索引 + 配置，返回结果 |
| **配置优先级链** | 命令行参数 > `pyproject.toml [tool.simulation]` > 代码默认值 |
| **IO 容错** | 所有文件写入操作均捕获 OSError，输出警告而非崩溃 |
| **纯 Python** | 不依赖 numpy（可选），全 Python 标准库实现（除 psutil + matplotlib） |

                        ┌──────────────────────┐

## 3. 模拟逻辑

### 3.1 宇宙模型

#### 3.1.1 环形拓扑（Toroidal）

宇宙是一个**正方形平面**，边长由 `universe_size` 指定（默认 10000 光年）。

采用**环形拓扑**：物体从右边界穿出即从左边界进入，从上边界穿出即从下边界进入。

```
          ┌──────────────────────┐
          │                      │
   ↑      │   ● A               │
   |      │                      │
   |      │          ● B        │
   y      │                      │
          │               ● C    │
          │                      │
          └──────────────────────┘
          ────────────→
                x
```

坐标归一化函数 `normalize_position(x, y, size)` 确保任何坐标都在 `[0, size) × [0, size)` 范围内。距离计算 `ring_distance(x1, y1, x2, y2, size)` 使用环形最短路径。

#### 3.1.2 空间索引（Spatial Index）

为加速文明间距离查询，采用**网格索引**（Grid Spatial Index）：

- 将宇宙空间划分为 `cell_size × cell_size` 的网格
- 每个网格存放落入该区域的文明 ID 列表
- 查询时仅搜索目标网格及其 8 邻域网格
- `cell_size` 可自动根据文明数量和平均探测范围计算

### 3.2 文明生命周期

每个文明 (`Civilization`) 经历以下阶段：

```
诞生 (birth)
  │
  ▼
技术发展 (development) ←──────────────────┐
  │                                         │
  ▼                                         │
空间扩张 (expansion)                        │
  │                                         │
  ▼                                         │
探测与接触 (detection)                      │
  │                                         │
  ├─ 发现其他文明 → 威胁评估               │
  │    ├─ 威胁高 → 主动攻击 (attack)       │
  │    ├─ 威胁低 → 隐匿 (stealth)          │
  │    └─ 中等 → 暴露坐标 (exposure)       │
  │                                         │
  ▼                                         │
存活 (alive) ────── 进入下一时间步 ────────┘
  │
  ├─ 被攻击消灭 → is_alive = False → 清理
  ├─ 宇宙打击 → is_alive = False → 清理
  └─ 自然存活 → 继续演化
```

**关键属性**（`Civilization` 数据类）：

| 属性 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `id` | int | — | 唯一标识 |
| `name` | str | — | 自动生成名称 |
| `x, y` | float | [0, size) | 空间坐标 |
| `level` | int | 1+ | 文明等级，越高越强 |
| `tech_points` | float | 0+ | 科技点数，决定等级升级 |
| `expansion_radius` | float | 0+ | 扩张半径（影响力范围） |
| `population` | float | 0+ | 人口规模 |
| `energy_output` | float | 0+ | 能量输出水平 |
| `aggressiveness` | float | [0, 1] | 攻击性，影响攻击决策 |
| `stealth` | float | [0, 1] | 隐蔽性，影响被探测难度 |
| `detection_range` | float | 0+ | 探测距离（光年） |
| `is_alive` | bool | — | 是否存活 |
| `birth_time` | int | — | 诞生的时间步 |
| `communication_active` | bool | — | 是否正在主动通信（暴露状态） |

### 3.3 单步 10 阶段流程

`Simulation.step()` 按严格顺序执行以下 10 个阶段：

#### Phase 1: 重建空间索引
将所有存活文明重新映射到空间索引网格中。每次步进都要重建，因为文明位置会因扩张而改变。

#### Phase 2: 文明诞生 (Birth)
根据当前存活文明数量计算新文明诞生数：`new_births = alive_count × birth_rate × random`。新文明在宇宙中随机位置生成，拥有基础的随机参数。

#### Phase 3: 技术发展 (Development)
每个文明：
1. 科技点自然增长：`tech_points += tech_growth_base`
2. 人口增长：`population ×= (1 + pop_growth_rate)`
3. 能量增长：`energy_output ×= (1 + energy_growth_rate)`
4. **技术爆炸判定**：若 `tech_points` 达到等级升级所需值，触发技术爆炸
   - 等级 +1
   - 探测范围、扩张半径、攻击性全面提升
   - 携带能力（Carrying Capacity）限制人口增长

#### Phase 4: 扩张 (Expansion)
每个文明以 `expansion_radius` 为半径扩张。每步：
- 扩张半径增长：`expansion_radius += expansion_rate_base`
- **坐标暴露判定**：若扩张半径超过 `exposure_threshold`，文明坐标暴露概率上升
- 暴露的文明 `communication_active = True`，更容易被其他文明探测到

#### Phase 5: 探测与接触 (Detection)
使用空间索引查询每个文明在 `detection_range` 内的邻居文明。
- 探测概率受目标文明 `stealth` 和 `communication_active` 状态影响
- 探测成功即生成 `ContactEvent`，记录双方文明、距离和接触时间
- 所有接触事件被返回给 Phase 6 处理

#### Phase 6: 黑暗森林 (Dark Forest)
对每个 `ContactEvent`，计算**威胁感知**：

```
threat = (target.level / observer.level) × 0.4
       + (target.aggressiveness) × 0.3
       + (target.tech_points / max(observer.tech_points, 1)) × 0.3
```

决策逻辑：
| 威胁值 | 行为 | 说明 |
|--------|------|------|
| ≥ 0.65 | 攻击 | 先发制人，尝试消灭目标 |
| 0.35 ~ 0.65 | 暴露 | 仅在接触中暴露坐标 |
| ≤ 0.35 | 隐匿 | 保持沉默，不暴露 |

**攻击判定**：攻击成功概率 = `attacker.tech_points / (attacker.tech_points + defender.tech_points)`。若成功，防御方 `is_alive = False`。

#### Phase 7: 宇宙打击 (Cosmic Strike)
全局低概率事件：每步以 `cosmic_strike_prob` 概率触发。
- 打击中心随机选取
- 打击半径 = `universe_size × random.uniform(0.02, 0.1)`
- 使用 `query_region` 查找打击范围内的所有文明并毁灭

#### Phase 8: 清理死亡文明
从 `self.civilizations` 中移除所有 `is_alive == False` 的文明。

#### Phase 9: 统计收集与数据保存
- `StatsCollector.collect()` 计算本步统计（文明数、等级分布、平均科技等）
- `DataSaver.save_step()` 写入 CSV 步文件 + 追加汇总行

#### Phase 10: 推进步数 + 回调
- `current_step += 1`
- 调用所有已注册的 `_on_step_callbacks`

### 4.4 `src/rules/tech_bomb.py` — 技术爆炸规则

| 函数 | 说明 |
|------|------|
| `apply_development(civilizations, config)` | 对所有存活文明执行技术发展：科技增长、人口增长、能量增长、等级升级和技术爆炸 |
| `_tech_needed(level) -> float` | 计算升级到下一级所需科技点数（指数增长：`100 × 2^level`） |
| `_calc_carrying_capacity(civ) -> float` | 计算文明承载力上限（科技水平 × 扩张半径），限制人口过度增长 |
| `_trigger_tech_explosion(level, ...) -> (new_level, new_tech_prob)` | 触发技术爆炸：等级跳跃提升 + 探测/扩张全面增强 |

**技术爆炸效果**：触发时文明等级跳跃式提升（`level += random(1, 3)`），同时：
- 科技点数大幅提升
- 探测范围倍增
- 扩张半径倍增
- 下次技术爆炸概率降低

### 4.5 `src/rules/expansion.py` — 扩张规则

| 函数 | 说明 |
|------|------|
| `apply_expansion(civilizations, config)` | 对所有存活文明执行扩张：增长扩张半径 + 坐标暴露判定 |

**扩张逻辑**：
- `expansion_radius += expansion_rate_base`
- 若 `expansion_radius > exposure_threshold`，则按 `base_exposure_prob` 暴露坐标
- 暴露程度与扩张半径成正比：`exposure_prob = min(base_exposure_prob × (expansion_radius / exposure_threshold), 1.0)`
- 暴露后 `communication_active = True`

### 4.6 `src/rules/detection.py` — 探测规则

#### `class ContactEvent`
| 属性 | 说明 |
|------|------|
| `civ_a: Civilization` | 发起探测的文明 |
| `civ_b: Civilization` | 被发现的文明 |
| `distance: float` | 两者之间的距离 |
| `step: int` | 发现时的时间步 |

#### 顶层函数
| 函数 | 说明 |
|------|------|
| `detect_contacts(civilizations, spatial_index, config) -> list[ContactEvent]` | 查询每个文明探测范围内是否有其他文明，返回所有接触事件 |

**探测判定**：
- 使用 `spatial_index.query_neighbors()` 获取探测范围内的邻居
- 仅处理 `civ_a.id < civ_b.id` 避免重复
- 探测成功概率受目标 `stealth` 影响：`prob = 1.0 - target.stealth`
- 目标处于 `communication_active` 状态时，探测概率翻倍

### 4.7 `src/rules/dark_forest.py` — 黑暗森林规则

| 函数 | 说明 |
|------|------|
| `_calculate_threat(observer, target) -> float` | 计算观察者对目标的威胁感知值（等级比 × 攻击性 × 科技比） |
| `_decide_action(threat, config) -> str` | 根据威胁值决定行为："attack" / "expose" / "hide" |
| `_attempt_attack(attacker, defender) -> bool` | 尝试攻击：成功概率 = tech_points 比，成功后毁灭目标 |
| `_expose_civilization(civ)` | 暴露文明坐标（设置 communication_active = True） |
| `apply_dark_forest(civilizations, contacts, config) -> (int, int)` | 对接触事件执行黑暗森林判定，返回 (攻击次数, 毁灭次数) |
| `apply_cosmic_strike(civilizations, config) -> int` | 全局宇宙打击，返回被毁灭的文明数量 |

**黑暗森林流程**：
```
每个 ContactEvent:
  1. 计算 observer 对 target 的威胁
  2. 根据威胁值决定行为：
     - attack: 尝试攻击 target（可能成功或失败）
     - expose: 暴露 observer 自身坐标
     - hide: 无操作
  3. 计算 target 对 observer 的威胁（对称处理）
```

**宇宙打击流程**：
```
每步以 cosmic_strike_prob 概率触发:
  1. 随机选择打击中心 (x, y)
  2. 打击半径 = universe_size × random(0.02, 0.1)
  3. 查询打击范围内所有文明
  4. 范围内文明全部毁灭
```

**CLI 参数一览**：

| 参数 | 类型 | 说明 |
|------|------|------|

| `--load FILE` | str | 从 JSON 状态文件恢复 |
| `--save-interval N` | int | 保存间隔 |
| `--detect-only` | flag | 仅检测性能，不运行模拟 |
| `--name-mode {auto,number}` | str | 名称生成模式 |
| `--seed N` | int | 随机种子（可重现模拟） |

**main() 调度流程**：
```
main(argv)
  ├─ --detect-only → 检测性能→打印报告→return 0
  ├─ --batch → BatchRunner(config).run_all() → 打印+保存
  └─ 其他：
       1. load_config(args) 合并配置
       2. 设置随机种子（--seed）
       3. --load → Simulation.load_state(file)
       4. 否则 → Simulation(config).initialize()
       5. 分发：
          ├─ --fast → run_fast(sim)
          ├─ --interactive → run_interactive(sim)
          └─ 默认 → run_standard(sim)
```
