# 宇宙点状文明体系模拟软件 — 需求文档

## 1. 项目概述

### 1.1 目标
开发一款基于 Python 的模拟软件，将宇宙间的智慧文明抽象为二维平面上的"点"，每个点由一组参数描述。通过离散时间步推进，模拟大量文明在"黑暗森林"法则下的诞生、演化、接触与毁灭过程。

### 1.2 灵感来源
刘慈欣《三体 II：黑暗森林》中的宇宙社会学设定。

---

## 2. 核心设定

### 2.1 宇宙公理（软件中作为底层规则实现）
1. **第一公理**：生存是文明的第一需要。
2. **第二公理**：文明不断增长和扩张，但宇宙中的物质总量保持不变。

### 2.2 黑暗森林机制（全部纳入）
| 机制 | 说明 |
|------|------|
| **猜疑链** | 两个文明接触时，双方无法判断对方是否友好，默认选择攻击 |
| **技术爆炸** | 低级文明可能在短期内实现技术飞跃，快速超越高级文明 |
| **打击即毁灭** | 高级文明发现低级文明后可直接发动攻击将其毁灭 |
| **暴露即死亡** | 文明一旦被其他文明发现位置，就面临被毁灭的风险 |
| **黑暗森林打击** | 存在"二向箔/光粒"等宇宙公理级打击手段 |
| **坐标暴露** | 文明在通信或扩张时可能暴露自身坐标 |

---

## 3. 文明参数设计

每个文明由以下参数描述（~15 个参数）：

| 编号 | 参数名 | 类型 | 说明 |
|------|--------|------|------|
| 1 | `id` | int | 唯一标识 |
| 2 | `name` | str | 文明名称（自动生成或预设） |
| 3 | `x` | float | 二维空间 X 坐标 |
| 4 | `y` | float | 二维空间 Y 坐标 |
| 5 | `level` | int | 文明等级（1~5） |
| 6 | `tech_points` | float | 科技点数（用于触发技术爆炸） |
| 7 | `tech_explosion_prob` | float | 技术爆炸概率 |
| 8 | `expansion_radius` | float | 扩张半径（文明影响力范围） |
| 9 | `population` | float | 人口规模 |
| 10 | `energy_output` | float | 能量输出水平 |
| 11 | `aggressiveness` | float | 攻击性（0~1，越高越倾向主动攻击） |
| 12 | `stealth` | float | 隐蔽性（0~1，越高越难被探测） |
| 13 | `detection_range` | float | 探测范围（能发现多远内的其他文明） |
| 14 | `is_alive` | bool | 是否存活 |
| 15 | `birth_time` | int | 诞生时间步 |
| 16 | `communication_active` | bool | 是否正在主动通信（暴露风险） |

> 注：参数精确数量和名称在开发阶段可能微调。

---

## 4. 文明数量

初始规模：**几千个文明**（如 3,000 ~ 10,000）

系统应支持：
- 配置初始文明数量
- 模拟过程中新文明不断诞生
- 模拟过程中文明不断被毁灭

---

## 5. 模拟引擎

### 5.1 时间推进方式
**离散时间步**，每步代表一个宇宙时间单位（如 100 年/步），每步执行完整的演化逻辑。

### 5.2 每步演化逻辑
在每个时间步内，按顺序执行以下流程：

```
1. 文明诞生阶段
   - 按概率在宇宙空域随机生成新文明
   - 新文明初始参数根据宇宙平均发展水平随机生成

2. 文明发展阶段
   - 每个文明根据自身参数推进内政发展
   - 科技点数自然积累
   - 人口增长
   - 能量输出增长
   - 技术爆炸判定（若触发，level 跳升，属性暴涨）

3. 扩张阶段
   - 文明根据等级和能量水平向外扩张影响力范围
   - 扩张可能暴露自身坐标

4. 探测与接触阶段
   - 每个文明在其探测范围内搜索其他文明
   - 若发现其他文明，触发"接触事件"

5. 接触处理阶段（猜疑链）
   - 接触双方根据自身攻击性、双方等级差、隐蔽性等参数
   - 判定是否互相攻击、一方攻击另一方、或互相规避
   - 攻击方成功则毁灭目标文明
   - 被攻击方可能反击

6. 黑暗森林打击阶段
   - 极低概率触发宇宙公理级打击事件
   - 打击可毁灭大片区域内的所有文明

7. 清理阶段
   - 移除所有已毁灭的文明
   - 统计本步数据
```

### 5.3 性能要求
软件应提供**自动检测当前电脑计算水平**的能力，并根据检测结果给出推荐的模拟参数（如文明数量上限、每步计算耗时预估等）。

## 6. 运行模式（三种）

| 模式 | 名称 | 可视化 | 交互性 | 速度 |
|------|------|--------|--------|------|
| **高性能模式** | `--fast` | 无（仅终端文本输出） | 无 | 最快 |
| **标准模式** | `--standard` | 实时 matplotlib 图表 | 无 | 中等 |
| **交互模式** | `--interactive` | 实时 matplotlib 图表 + 图形界面按钮 | 可暂停/调速/查看文明细节 | 较慢（用户可控） |

用户通过命令行参数选择运行模式：
```bash
# 高性能模式
uv run python main.py --fast

# 标准模式
uv run python main.py --standard

# 交互模式
uv run python main.py --interactive
```

## 7. 输出与可视化

所有模式下**始终保存数据文件**。

### 7.1 终端文本输出
每步输出概要统计，例如：
```
时间步: 100 | 存活文明: 4,238 | 新生: 12 | 毁灭: 8 | 最高等级: 4 | 平均等级: 2.1
```

### 7.2 数据文件保存
自动保存至 `output/` 目录：
- **分步全量数据**：每步所有存活文明的完整参数（CSV 格式）
  - 文件名：`step_000001.csv`, `step_000002.csv`, ...
- **汇总统计**：每步的统计数据汇总（CSV 格式）
  - 文件名：`summary.csv`
  - 包含：时间步、文明总数、新生数、毁灭数、等级分布、平均科技水平等

### 7.3 实时图表（标准模式和交互模式）
- **左图**：文明二维空间分布散点图（颜色表示等级，大小表示能量）
- **右图**：文明数量/等级分布等统计曲线的实时更新图
- **可选**：历史趋势图

### 7.4 交互模式图形界面按钮
- ▶ **开始/暂停**（快捷键：空格）
- ⏩ **加快速度**（快捷键：+）
- ⏪ **减慢速度**（快捷键：-）
- ⏭ **单步执行**（快捷键：→）
- 🔍 **查看文明详情**（点击散点图中的文明点，弹出信息窗口）
- 💾 **导出当前状态**
- ❌ **退出**（快捷键：Esc 或 Ctrl+Q）
